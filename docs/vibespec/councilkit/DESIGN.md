# CouncilKit - 设计规约

## 信息架构

- 应用外壳（侧边栏 + 主内容区）
  - 侧边栏
    - 新建房间入口
    - 房间列表（按最近活跃排序）
    - Agent 模板入口（P1）
  - 主内容区
    - 房间讨论视图
      - 话题头部
      - 讨论流（时间线 / 分栏对比可切换）
      - 每轮总结区块
      - 用户输入与发起控制
    - 空状态视图（无房间 / 未选中房间）
    - Agent 模板管理视图（P1）

## 页面清单

### 页面 1: 主页 / 空状态

- 路由: /
- 用途: 应用入口。有房间时重定向到最近活跃房间;无房间时展示空状态引导
- 关键元素: "新建房间"主操作按钮、产品一句话说明
- 进入条件: 打开 app、点击侧边栏 logo
- 离开路径: → /rooms/new（新建）、→ /rooms/:roomId（点击列表房间）

### 页面 2: 新建房间

- 路由: /rooms/new
- 用途: 一站式完成话题输入 + agent 配置 + 轮数设定 + 发起讨论,满足 SC-1（30 秒内）
- 关键元素: 话题输入框、agent 配置卡片列表（加号添加）、讨论轮数选择、"发起讨论"按钮
- 进入条件: 侧边栏"新建房间"
- 离开路径: 发起讨论后 → /rooms/:roomId；取消 → 返回上一页

### 页面 3: 房间讨论

- 路由: /rooms/:roomId
- 用途: CouncilKit 核心界面。展示多 agent 讨论流、每轮总结,支持用户参与、追问、视图切换、中途增减 agent
- 关键元素: 话题头部、视图切换控件（时间线/分栏对比）、讨论流、轮次分隔与进度、每轮总结折叠块、用户输入框、agent 管理入口
- 进入条件: 点击房间列表、新建房间后跳转
- 离开路径: → 其他房间、→ /rooms/new、→ /templates

### 页面 4: Agent 模板管理（P1）

- 路由: /templates
- 用途: 创建、查看、删除可复用的 agent 配置组合
- 关键元素: 模板列表、新建模板、模板内 agent 明细
- 进入条件: 侧边栏"Agent 模板"
- 离开路径: → 房间、新建房间时引用模板

## 组件树

### 共享组件

- AppShell: 应用外壳,左侧边栏 + 右主内容区两栏布局。职责:全局导航骨架
- Sidebar: 侧边栏容器。职责:承载新建入口、房间列表、模板入口
- RoomListItem: 房间列表项。变体:活跃/普通/有未读总结。使用位置:Sidebar
- AgentConfigCard: agent 配置卡片。职责:选模型 + 定角色。变体:可编辑/只读。使用位置:新建房间、agent 管理面板、模板管理
- ModelSelect: 模型下拉选择（Claude / DeepSeek / GPT）。使用位置:AgentConfigCard
- MessageBubble: 单条发言气泡。变体:agent 发言 / 用户发言 / 离线标注。职责:展示发言者归属与内容
- TypingIndicator: 打字指示器。职责:标识"某 agent 正在发言"
- SummaryBlock: 总结折叠块。变体:展开/折叠。职责:展示某一轮的总结摘要
- EmptyState: 通用空状态。变体:无房间 / 无模板 / 房间无讨论
- ErrorBanner: 错误提示条。职责:展示模型超时、网络断开等错误与重试入口
- Button / TextInput / Textarea: 基础控件

### 页面级组件

#### 新建房间（/rooms/new）

- NewRoomForm: 新建房间表单容器
  - TopicInput: 话题输入框（TextInput）
  - AgentConfigList: agent 配置列表
    - AgentConfigCard（共享）
    - AddAgentButton: 添加 agent
    - TemplatePicker: 引用已有模板（P1）
  - RoundCountSelect: 讨论轮数选择
  - StartDiscussionButton: 发起讨论

#### 房间讨论（/rooms/:roomId）

- RoomView: 房间视图容器
  - RoomHeader: 话题头部 + agent 管理入口 + 视图切换控件
    - ViewToggle: 时间线 / 分栏对比切换
    - AgentManagePanel: 中途加入/移除 agent（P1）
  - DiscussionStream: 讨论流容器
    - TimelineView: 时间线视图
      - RoundDivider: 轮次分隔与进度指示
      - MessageBubble（共享）
      - TypingIndicator（共享）
      - SummaryBlock（共享,每轮一个）
    - ColumnCompareView: 分栏对比视图（P1,R10）
      - AgentColumn: 单 agent 列
  - UserInputBar: 用户输入与追问
    - Textarea（共享）
    - SendButton: 发送/追问,触发新一轮

#### Agent 模板管理（/templates）

- TemplateListView: 模板列表
  - TemplateCard: 单个模板
  - NewTemplateButton

## 核心用户流程

### 流程 1: 快速发起一次多 agent 讨论（对应场景 1）

1. 用户点击侧边栏"新建房间" → 进入 /rooms/new
2. 用户输入讨论话题 → 话题填入 TopicInput
3. 用户点击"添加 agent",选模型 + 填角色（可重复 2-3 次）→ AgentConfigCard 列表增长
4. 用户设定讨论轮数（默认 2 轮）→ RoundCountSelect
5. 用户点击"发起讨论" → 跳转 /rooms/:roomId,首个 agent 进入 typing 状态
6. agent 按顺序依次流式发言,后者引用前者 → 时间线逐条出现
7. 本轮全部发言完成 → 自动生成本轮 SummaryBlock 并展开
8. 达到设定轮数 → 讨论暂停,等待用户追问或结束

### 流程 2: 在已有房间中继续深入讨论（对应场景 2）

1. 用户点击侧边栏房间列表项 → 进入 /rooms/:roomId,展示历史讨论与各轮总结
2. 用户在 UserInputBar 输入追问 → 点击发送
3. 系统基于已有上下文触发新一轮 → agent 依次回应,引用历史
4. 新一轮结束 → 生成新的 SummaryBlock
5. 上下文过长时 → 系统自动压缩历史(对用户不可见,延后到 Tech)

### 流程 3: 用户作为参与者发言（对应 R8）

1. 讨论进行中或某轮结束后,用户在 UserInputBar 输入观点
2. 用户发言以"用户气泡"插入时间线,身份与 agent 区分
3. 下一轮 agent 发言时能看到并回应用户观点

### 流程 4: 切换到分栏对比看独立回答（对应 R10,P1）

1. 用户在 RoomHeader 点击 ViewToggle → 切到 ColumnCompareView
2. 每个 agent 的回答按列并排展示,便于对比
3. 用户可切回时间线继续讨论

## 交互状态

### DiscussionStream（讨论流）

| 状态 | 触发条件 | 显示内容 | 用户可执行操作 |
|------|----------|----------|----------------|
| 加载中 | 发起讨论后、首个 agent 尚未输出 | TypingIndicator + "X 正在思考" | 等待;可取消本轮 |
| 流式输出 | agent 正在逐字发言 | 当前气泡逐字出现,其余 agent 等待 | 阅读;可插入发言 |
| 空状态 | 房间刚创建、尚无任何讨论 | "还没有讨论,在下方输入开始" | 在 UserInputBar 输入 |
| 错误 | 某模型 API 超时/不可用 | 该 agent 标"离线",ErrorBanner 提示,其他 agent 继续 | 重试该 agent / 忽略继续 |
| 正常 | 一轮或多轮讨论完成 | 完整时间线 + 各轮 SummaryBlock | 追问、切换视图、增减 agent |
| 边界:全部 agent 离线 | 所有模型均不可用 | 整体 ErrorBanner + "稍后重试" | 重试 / 检查配置 |

### NewRoomForm（新建房间）

| 状态 | 触发条件 | 显示内容 | 用户可执行操作 |
|------|----------|----------|----------------|
| 加载中 | 提交发起讨论、跳转前 | 按钮 loading | 等待 |
| 空状态 | 刚进入、未填话题/未加 agent | 占位提示,发起按钮禁用 | 填话题、加 agent |
| 错误 | 话题为空 / 未配置 agent 就点发起 | 行内校验提示 | 补全必填项 |
| 正常 | 话题非空且至少 1 个 agent | 发起按钮可用 | 发起讨论 |

### Sidebar 房间列表

| 状态 | 触发条件 | 显示内容 | 用户可执行操作 |
|------|----------|----------|----------------|
| 空状态 | 无任何房间 | "还没有房间" + 新建引导 | 新建房间 |
| 加载中 | 首次加载房间列表 | 骨架占位 | 等待 |
| 正常 | 有房间 | 房间列表,最近活跃在上 | 选择房间、新建 |

## 数据流向

### 数据实体

- 房间（Room）: 房间标识、话题、创建时间、最近活跃时间、关联 agent 列表、讨论轮次列表
- Agent: agent 标识、底层模型、角色/立场、在线状态、所属房间或模板
- 发言（Message）: 发言标识、发言者（agent 或用户）、内容、所属轮次、时间
- 轮次（Round）: 轮次序号、本轮发言集合、本轮总结
- 总结（Summary）: 关联轮次、摘要内容、生成时间
- Agent 模板（Template,P1）: 模板名、agent 配置组合

### 数据流转

- 新建房间: 表单数据 → 创建 Room + 关联 Agent → 进入房间视图
- 发起讨论: Room 上下文 + Agent 列表 → 按序产生 Message → 填充当前 Round → 触发 Summary
- 追问: 用户 Message + 历史上下文 → 新 Round → 新 Message 集 → 新 Summary
- 用户发言: 用户 Message 插入当前 Round → 下一轮 agent 读取
- 视图切换: 同一份 Message/Round 数据,时间线视图按时间排列,分栏视图按 agent 分组
- 持久化: Room / Message / Round / Summary 落地存储,房间列表与历史回看从存储读取（方案延后到 Tech）

## 视觉方向

### 整体调性

深色为主的专业工具界面,信息密集但层级清晰,接近 Linear 与 TypingMind 的工作台观感——为长时间阅读多 agent 多轮讨论优化。

### 配色方案

- 主色: 蓝紫强调色（参考 Linear,约 #5E6AD2）
- 背景: 深色为主（主背景约 #0E0E10,面板约 #1A1A1E）
- 强调: 蓝紫主色用于主操作按钮、当前发言高亮
- 语义色: 成功=绿（约 #3FB950）、警告=琥珀（约 #D29922）、错误=红（约 #F85149）、信息=蓝（约 #58A6FF）
- agent 区分: 每个 agent 分配一个固定强调色 + 头像首字母,区分不仅靠颜色还靠头像与名称（可访问性要求）

### 排版

- 字体风格: 无衬线为主（正文/UI）,等宽字体用于 agent 引用前文的代码/被引片段
- 大小层级: 三级标题（话题 H1、轮次/区块 H2、agent 名 H3）+ 正文 + 辅助文字（时间戳、状态）

### 间距与布局

- 间距基准: 4px 栅格（4 / 8 / 12 / 16 / 24）
- 布局模式: 侧边栏（约 260px）+ 主内容区自适应
- 最大内容宽度: 讨论流主列约 760px 居中,保证可读行宽

### 视觉参考

- Linear: 借鉴深色面板、强调色用法、信息密度与侧边栏导航
- TypingMind: 借鉴侧边栏对话列表 + 主聊天区结构、多模型切换的呈现
- Discord/Slack: 借鉴消息流中发言者归属（头像 + 名称 + 颜色）的清晰标识

### 视觉约束

- 不做: 不用大面积渐变、不用拟物化、不用花哨入场动画;agent 区分不得仅依赖颜色

## 响应式要求

| 断点 | 宽度范围 | 布局变化 |
|------|----------|----------|
| 桌面 | ≥ 1024px | 侧边栏常驻 + 主内容区,完整体验（主目标） |
| 平板 | 640–1023px | 侧边栏可收起为抽屉,主内容区占满;分栏对比最多 2 列 |
| 移动端 | < 640px | 侧边栏改为顶部抽屉;分栏对比退化为时间线（不并排) |

注:PRD 定位为独立开发者/知识工作者的桌面工具,桌面为一等公民,平板/移动为适配。

## 可访问性要求

- A11y-1: 多 agent 发言区分不仅靠颜色,同时用头像首字母 + 名称标签
- A11y-2: 讨论流、视图切换、输入框、房间列表均支持键盘 Tab 导航与回车操作
- A11y-3: 正文与背景对比度满足 WCAG AA（深色背景下正文对比度 ≥ 4.5:1）
- A11y-4: 流式输出与 typing 状态对屏幕阅读器有 aria-live 提示（具体实现延后到 Tech）

## 设计决策记录

| 决策 | 选择 | 放弃的替代方案 | 原因 |
|------|------|----------------|------|
| 核心讨论 UI 范式 | 聊天时间线为主 | 分栏对比为主 / 一开始就混合双视图 | R4 互相质疑是 P0,时间线最贴合对话感;分栏作为 P1 可切换视图覆盖 R10 |
| 讨论结束定义 | 轮数驱动 | 用户手动结束 / 单轮+手动继续 | 总结触发时机明确、可预测,agent 不会无限刷屏,契合 SC-4 可控延迟 |
| 同轮发言顺序 | 依次顺序发言 | 并行流式发言 | 后者能看到前者发言才能成立 R4;首条可快速流式出现满足 SC-4 |
| 总结呈现 | 每轮结束嵌入时间线的折叠块 | 固定底部总结栏 / 单独总结页 | 总结紧跟讨论上下文,最新一轮默认展开,符合 A4 降低认知负担 |
| 视觉调性 | 深色工具风 | 亮色简约 / 亮暗双主题 | 匹配桌面专业工具定位与长时间阅读;MVP 先做一套主题降低成本 |
| 导航模式 | 侧边栏 + 主内容区 | 顶部标签 / 全屏单房间 | 同类产品通用范式,房间列表与讨论可并存,支持 R12 历史回看 |

## 延后到 Tech

- 多模型 API 并发/串行调用与首条 ≤10 秒的延迟控制实现策略（R7 / SC-4）
- 长对话上下文压缩/截断的具体算法（场景 2 边界）
- 房间与讨论历史的本地数据存储方案
- Web app vs macOS native 技术选型(影响窗口/抽屉等具体实现)
- 流式输出与 aria-live 的具体技术实现（A11y-4）
- 总结生成由哪个模型/机制负责
